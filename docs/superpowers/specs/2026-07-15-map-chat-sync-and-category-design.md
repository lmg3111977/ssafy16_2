# LocalHub 범주 지도와 챗봇 추천 연동 설계

## 목표

기존 축제 전용 Leaflet 지도를 제공 JSON 기반의 범주별 지역정보 지도로 확장하고, 챗봇 답변에 좌표가 있는 추천이 포함되면 답변 아래의 `지도에서 보기` 버튼으로 같은 지도에 표시한다. 기존 축제 필터, 챗봇 답변, 커뮤니티 기능은 유지한다.

## 개발 의뢰서 조건 준수

- Vue.js 3와 Vite 기반의 단일 SPA 구조를 유지한다.
- 지도는 기존 Leaflet을 사용하며 새 지도 API나 지도 라이브러리를 추가하지 않는다.
- 지도 탐색 데이터는 제공된 서울 JSON을 프론트엔드에서 직접 불러와 처리한다.
- 외부 데이터를 추가 수집하지 않으므로 별도 라이선스 검토 대상이 생기지 않는다.
- 기존 Netlify 배포 구조와 OpenAI 챗봇 API 계약을 유지한다.
- 커뮤니티의 localStorage CRUD 동작과 저장 형식은 변경하지 않는다.
- `.env`와 비밀값을 수정하거나 브라우저에 노출하지 않는다.

## 확정된 사용자 경험

### 범주 지도

- 지도 위에 `축제·행사`, `관광지`, `문화시설`, `레포츠`, `숙박`, `여행 코스` 범주 버튼을 표시한다.
- 한 번에 한 범주만 선택할 수 있다.
- 기본 범주는 `축제·행사`다.
- 쇼핑 데이터는 4,368건으로 다른 범주보다 커서 전체 탐색 범주에서는 제외한다.
- 쇼핑 데이터와 챗봇의 쇼핑 질의 기능 자체는 삭제하지 않는다.
- 자치구 필터는 모든 범주에 적용한다.
- 시작일과 종료일 필터는 날짜 필드가 있는 `축제·행사` 범주에서만 표시한다.
- 결과 건수는 중복 제거와 유효 좌표 검사를 통과한 실제 지도 표시 건수를 사용한다.

### 챗봇 추천 지도

- 챗봇 답변의 출처 중 유효한 좌표가 한 건 이상 있을 때만 답변 아래에 `지도에서 보기` 버튼을 표시한다.
- 버튼은 추천 카드마다 두지 않고 답변 하나당 하나만 표시한다.
- 버튼을 누르면 해당 답변의 유효한 추천 장소를 한꺼번에 지도에 표시한다.
- 질문과 답변의 실제 출처 유형에 따라 축제, 관광지, 문화시설, 레포츠, 숙박, 여행 코스, 쇼핑을 표시한다.
- 쇼핑은 전체 탐색 범주에서는 제외하지만 챗봇이 반환한 소수의 추천 장소는 표시한다.
- 복합 질문으로 여러 유형이 추천되면 그 유형들을 함께 표시한다.
- 지도는 추천 마커가 모두 보이도록 자동 확대하고 지도 영역으로 부드럽게 이동한다.
- `기존 지도로 돌아가기`를 누르면 챗봇 지도 모드를 종료하고 마지막 선택 범주와 필터 상태를 복원한다.

## 아키텍처

`App.vue`가 챗봇과 지도 사이의 명시적인 상태 연결을 담당한다. 브라우저 전역 이벤트나 새 전역 상태 라이브러리는 사용하지 않는다.

```text
FestivalChatWidget
  -> map-request 이벤트(LocalHubSource[])
App.vue
  -> chatMapPlaces 상태 갱신
  -> FestivalMapSection에 추천 장소 전달
FestivalMapSection
  -> browse 또는 chat 모드 결정
  -> 필터·중복 제거·좌표 검증
FestivalMap
  -> 공통 MapPlace[] 마커 렌더링 및 화면 맞춤
```

이 구조는 데이터 흐름을 Vue props와 emits로 드러내므로 컴포넌트를 독립적으로 테스트할 수 있다.

## 공통 지도 자료형

기존 축제 전용 `Festival`을 지도 렌더러의 입력으로 직접 사용하지 않고, 지도에 필요한 필드만 가진 공통 `MapPlace`로 정규화한다.

```ts
type MapPlaceType =
  | 'festival'
  | 'attraction'
  | 'culture'
  | 'leisure'
  | 'accommodation'
  | 'shopping'
  | 'course'

interface MapPlace {
  id: string
  type: MapPlaceType
  title: string
  address: string | null
  latitude: number
  longitude: number
  imageUrl: string | null
  startDate: string | null
  endDate: string | null
  eventPlace: string | null
  fee: string | null
}
```

- 범주 JSON과 챗봇 `LocalHubSource`를 각각 `MapPlace`로 변환하는 함수를 분리한다.
- 지도 컴포넌트는 원본 JSON이나 챗봇 계약을 알지 않고 `MapPlace[]`만 받는다.
- 축제에만 있는 기간·행사장·요금 정보는 선택 필드로 유지한다.

## 데이터 로딩과 중복 제거

- 기본 축제 데이터는 현재처럼 초기 화면에서 준비한다.
- 다른 범주는 사용자가 처음 선택할 때 Vite 동적 import로 해당 JSON만 불러온다.
- 한 번 불러온 범주는 메모리에 캐시하여 재선택 시 다시 읽지 않는다.
- 제공 JSON 파일은 이동하거나 수정하지 않는다.
- 중복 키는 유효한 `contentid`를 최우선으로 사용한다.
- `contentid`가 없을 때만 정규화한 `제목 + 위도 + 경도` 조합을 대체 키로 사용한다.
- 중복 제거는 원본 데이터 순서를 유지하며 첫 항목을 남긴다.
- 숫자가 아니거나 서울 허용 범위를 벗어난 좌표는 지도 입력에서 제외한다.

## 지도 렌더링

- `FestivalMap.vue`는 `MapPlace[]`를 받는 공통 지도 컴포넌트로 확장한다.
- 범주 탐색 마커는 Leaflet의 가벼운 캔버스 기반 마커를 사용한다.
- 챗봇 추천 마커는 일반 범주 마커보다 눈에 띄는 강조 스타일을 사용한다.
- 마커 팝업에는 유형, 제목, 주소를 기본 표시하고 값이 있을 때만 이미지, 기간, 장소, 요금을 추가한다.
- 외부 데이터 문자열은 팝업 HTML에 넣기 전에 기존 방식으로 이스케이프한다.
- 범주나 필터가 바뀌면 유효 마커 전체가 보이도록 `fitBounds`를 적용한다.
- 결과가 없으면 서울 중심과 기본 확대 수준으로 돌아간다.

## 상태와 화면 흐름

`FestivalMapSection`은 다음 상태를 가진다.

- `mode`: `browse | chat`
- `selectedCategory`: 마지막으로 선택한 탐색 범주
- `selectedDistrict`: 탐색 모드의 자치구
- `selectedStartDate`, `selectedEndDate`: 축제 범주의 기간
- `categoryLoading`: 범주 JSON 로딩 상태
- `categoryError`: 범주 JSON 로딩 오류
- `chatPlaces`: 현재 챗봇 답변에서 전달된 지도 장소

범주 버튼이나 필터를 누르면 탐색 모드로 동작한다. 챗봇의 `지도에서 보기` 요청이 오면 필터 상태를 지우지 않고 채팅 모드만 활성화한다. 복귀 시 저장된 범주와 필터가 그대로 다시 나타난다.

## 오류 처리

- 유효 좌표가 없는 챗봇 답변에는 지도 버튼을 표시하지 않는다.
- 챗봇 출처 중 일부 좌표만 잘못된 경우 잘못된 항목만 제외하고 나머지를 표시한다.
- 범주 JSON 로딩에 실패하면 현재 마커를 유지하고 지도 위에 다시 시도 가능한 오류 문구를 표시한다.
- 지도 컴포넌트가 아직 준비되지 않았으면 추천 상태를 보존하고 마운트 후 렌더링한다.
- 팝업의 선택 정보가 없으면 항목을 숨기거나 `정보 없음`을 표시하며 예외를 발생시키지 않는다.
- 커뮤니티 유형은 좌표를 제공하지 않으므로 지도 대상에서 제외한다.

## 접근성

- 범주 버튼 그룹에는 명확한 레이블을 제공한다.
- 선택된 범주는 `aria-pressed`로 알린다.
- `지도에서 보기`와 `기존 지도로 돌아가기`는 실제 `button` 요소를 사용한다.
- 지도 이동 후 지도 섹션 제목에 프로그램 방식으로 초점을 옮겨 화면낭독기 사용자에게 상태 변경을 알린다.
- 결과 상태 문구에 현재 모드, 범주 또는 챗봇 추천, 표시 건수를 텍스트로 제공한다.
- `prefers-reduced-motion` 사용자는 부드러운 스크롤 대신 즉시 이동한다.

## 컴포넌트 변경 범위

- Modify: `src/App.vue` — 챗봇 지도 요청 상태와 props/emits 연결
- Modify: `src/chatbot/FestivalChatWidget.vue` — 답변 단위 지도 버튼과 `map-request` 이벤트
- Modify: `src/chatbot/types.ts` — 필요 시 지도 이벤트에 쓰는 기존 타입 재노출
- Modify: `src/features/map/FestivalMapSection.vue` — 범주, 데이터 로딩, 탐색·챗봇 모드, 필터 상태
- Modify: `src/features/map/components/FestivalMap.vue` — 공통 지도 장소와 마커 스타일 렌더링
- Modify: `src/features/map/types.ts` — `MapPlace`, 범주, 원본 JSON 타입
- Create: `src/features/map/data.ts` — 범주 로더, 정규화, 좌표 검증, 중복 제거
- Add tests: 지도 데이터 정규화, 챗봇 버튼·이벤트, 지도 섹션 상태 전환

백엔드 챗봇 응답에는 이미 `type`, `contentId`, `longitude`, `latitude`가 있으므로 `shared/chat-contract.ts`와 Netlify Function의 응답 형식은 변경하지 않는다.

## 검증

### 자동 검증

- 범주별 원본 JSON을 `MapPlace`로 올바르게 정규화한다.
- 중복 `contentid`는 한 건만 남긴다.
- 식별자 없는 중복은 제목·좌표 조합으로 제거한다.
- 잘못되거나 서울 밖인 좌표를 제외한다.
- 챗봇 답변에 유효 좌표가 있을 때만 지도 버튼을 표시한다.
- 버튼 클릭 시 해당 답변의 유효 출처만 `map-request`로 전달한다.
- 쇼핑 추천은 챗봇 지도에 포함한다.
- 범주 지도에는 쇼핑 선택 버튼이 없다.
- 범주 단일 선택, 축제 날짜 필터 표시, 비축제 날짜 필터 숨김을 확인한다.
- 챗봇 모드에서 탐색 모드로 복귀할 때 마지막 범주와 필터를 보존한다.
- 기존 챗봇, 커뮤니티, 검색 테스트를 포함한 `npm run verify`를 통과한다.

### 브라우저 검증

- 축제, 관광지, 문화시설, 레포츠, 숙박, 여행 코스 범주를 차례로 선택한다.
- 범주 변경 시 실제 표시 개수와 팝업 정보가 맞는지 확인한다.
- 자치구 필터가 모든 범주에 적용되는지 확인한다.
- 날짜 필터가 축제에서만 나타나는지 확인한다.
- 축제, 숙박, 복합, 쇼핑 질문의 `지도에서 보기` 동작을 확인한다.
- 챗봇 추천 지도에서 탐색 지도로 복귀했을 때 이전 상태를 확인한다.
- 데스크톱과 모바일에서 버튼, 스크롤, 지도 크기 재계산을 확인한다.

## 제외 범위

- 쇼핑 전체 데이터의 범주 지도 표시
- 여러 탐색 범주의 동시 선택
- 마커 클러스터링 라이브러리 설치
- 경로 탐색과 길찾기
- 새 공공데이터 또는 외부 API 수집
- 챗봇 검색 범위, 프롬프트, 모델 설정 변경
- 커뮤니티 기능과 저장 형식 변경
- 실제 Netlify 배포

## 완료 기준

- 사용자가 여섯 탐색 범주 중 하나를 선택해 중복 없는 유효 장소를 지도에서 볼 수 있다.
- 챗봇 답변에 지도 가능한 추천이 있으면 답변 아래 버튼으로 해당 추천을 지도에 표시할 수 있다.
- 쇼핑은 탐색 범주에서 제외되지만 챗봇의 소수 추천은 지도에 표시된다.
- 탐색 상태 복귀, 오류 처리, 접근성 상태가 정상 동작한다.
- 기존 프로젝트 필수 기능과 검증을 깨뜨리지 않는다.
