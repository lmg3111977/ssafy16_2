# 홈 대표 카테고리 및 챗봇 초기화 변경 디자인

## 목표

홈 화면의 데이터 현황 4칸을 서로 겹치는 전체 통계에서 사용자가 바로 이해할 수 있는 대표 탐색 카테고리로 변경한다.

## 표시 내용

왼쪽부터 다음 순서로 표시한다.

1. `783` — `관광지`
2. `201` — `축제·행사`
3. `566` — `문화시설`
4. `423` — `숙박`

숫자는 `netlify/functions/data`에 포함된 각 원본 JSON 파일의 `items` 개수를 기준으로 한다.

## 변경 범위

- `src/views/HomeView.vue`의 `stats` 배열만 수정한다.
- 현재 카드 레이아웃, 반응형 동작, 스타일은 유지한다.
- 카드를 링크나 버튼으로 바꾸지 않는다.
- 다른 카테고리 데이터, 지도 필터, 챗봇 답변 생성 방식은 변경하지 않는다.

## 챗봇 처음으로 돌아가기

- 첫 질문을 보낸 뒤 입력창 아래 안내 영역 왼쪽에 `↺ 처음으로` 버튼을 표시한다.
- 버튼을 누르면 진행 중인 답변 요청을 중단하고 기존 대화, 작성 중인 입력, 오류 상태를 지운다.
- 초기화 후 첫 환영 문구와 기존 빠른 질문 3개를 다시 표시한다.
- 챗봇 창은 열린 상태를 유지한다.
- 첫 화면에서는 버튼을 숨겨 불필요한 동작을 노출하지 않는다.

## 검증

- 홈 화면에 네 카테고리와 건수가 지정한 순서로 표시되는지 확인한다.
- 챗봇에서 첫 질문 이후 `처음으로` 버튼이 나타나는지 확인한다.
- 버튼을 누르면 환영 문구와 빠른 질문 3개가 다시 표시되는지 확인한다.
- 답변 생성 중 초기화해도 중단된 답변이나 오류가 뒤늦게 추가되지 않는지 확인한다.
- 타입 검사와 프로덕션 빌드가 통과하는지 확인한다.
- 데스크톱과 모바일에서 기존 4칸 레이아웃이 유지되는지 확인한다.
