# LocalHub 파란색 UI 리디자인 설계

## 목표

현재 프로젝트의 파란색 브랜드 인상과 모든 기능 로직을 유지하면서, 홈·지도·커뮤니티·챗봇의 시각적 완성도와 발표 흐름을 개선한다. 결과물은 Vue 3 + Vite 기반의 단일 정적 SPA로 유지한다.

## 확정 제약

- 프론트엔드는 Vue.js 3와 Vite를 유지한다.
- 결과물은 Netlify에 배포 가능한 단일 정적 SPA다.
- 화면 전환용 새 의존성이나 Vue Router를 설치하지 않는다.
- 현재 파란색 계열, 밝은 회색 배경, 흰색 카드의 브랜드 인상을 유지한다.
- 기존 Leaflet 지도, localStorage 커뮤니티, Netlify Functions 챗봇 로직을 변경하지 않는다.
- 커뮤니티 데이터 계약, 저장 형식, 입력 항목, CRUD 흐름을 변경하지 않는다.
- 챗봇 검색 범위, API 계약, 모델 설정, 오류 처리 로직을 변경하지 않는다.
- 새 데이터, 프레임워크, UI 라이브러리, 웹폰트를 추가하지 않는다.
- `netlify/functions/chat.mts`, `src/community/api.ts`, `src/components/CommunityBoard.vue`, `tests/community-storage.test.ts`의 현재 미커밋 작업을 되돌리지 않는다.

## 이번 구현 범위

이번 단계는 디자인과 레이아웃만 변경한다.

- 공통 색상·간격·반경·그림자 토큰 정리
- 공통 헤더와 화면 내비게이션 개선
- 홈 히어로와 기능 바로가기 재구성
- 지도 필터·지도·결과 영역의 시각적 위계 개선
- 기존 커뮤니티 작성 폼과 게시글 목록의 스타일 개선
- 기존 챗봇 위젯을 공통 디자인과 일치하도록 스타일 개선
- 데스크톱·태블릿·모바일 반응형 개선
- 키보드 포커스와 reduced-motion 등 시각 접근성 개선

다음 기능은 이번 단계에서 추가하지 않는다.

- 게시글 카테고리와 검색
- 게시글 상세·작성·비밀번호 모달
- 커뮤니티 저장 형식 변경
- 챗봇의 7종 데이터 통합 검색
- 챗봇 문맥·재시도·타임아웃 기능 변경
- 지도 데이터 유형 확장

## 정보 구조

하나의 Vue 앱에서 `home`, `explore`, `community` 세 화면을 내부 상태로 전환한다. 화면 전환은 디자인 셸의 일부이며, 기존 지도·커뮤니티·챗봇 컴포넌트는 그대로 재사용한다.

```text
App.vue
├─ SiteHeader
├─ HomeView
│  ├─ BlueHero
│  ├─ DatasetStats
│  └─ FeatureShortcuts
├─ ExploreView
│  └─ FestivalMapSection
├─ CommunityView
│  └─ CommunityBoard
├─ DataSourceFooter
└─ FestivalChatWidget
```

화면 상태는 URL 해시 `#home`, `#explore`, `#community`와 동기화한다. 새 HTML 문서나 서버 라우트는 만들지 않는다. 챗봇은 화면 전환과 관계없이 계속 유지한다.

## 시각 디자인

### 색상

- 기본 파랑: `#3165FF`
- 진한 파랑: `#2757E6`
- 제목 남색: `#15213A`
- 본문: `#4A5870`
- 보조 텍스트: `#778196`
- 페이지 배경: `#F7F9FC`
- 카드 배경: `#FFFFFF`
- 연한 파랑 배경: `#EDF2FF`
- 테두리: `#E4E9F1`
- 성공: `#1D6F31`
- 오류: `#9B2330`

현재 `#3165FF` 중심의 디자인을 유지하되, 큰 면에는 진한 남색과 연한 파랑을 조합하고 주황·금색 같은 새 브랜드 포인트는 사용하지 않는다.

### 타이포그래피

- 시스템 고딕 글꼴을 유지한다.
- 제목은 굵기 800~900, 본문은 400~600으로 위계를 만든다.
- 영문 대문자 레이블은 작은 크기와 넓은 자간으로 제한한다.
- 홈 제목은 `clamp()`를 사용하되 모바일에서 한글이 한 글자씩 끊기지 않도록 `word-break: keep-all`을 적용한다.

### 형태

- 주요 히어로: 반경 28~32px
- 일반 카드: 반경 18~24px
- 입력창: 반경 12~14px
- 주요 버튼: 반경 12~14px 또는 pill
- 그림자: 히어로와 플로팅 UI에는 중간 그림자, 일반 카드에는 약한 그림자
- hover는 최대 `translateY(-2px)`로 제한
- `prefers-reduced-motion`에서는 이동 애니메이션 제거

## 화면별 설계

### 공통 헤더

- 왼쪽에 기존 `LocalHub` 로고를 유지한다.
- 가운데에 `홈`, `지역 탐색`, `커뮤니티` 탭을 pill 컨테이너로 표시한다.
- 선택된 탭은 진한 파랑 배경과 흰색 글자로 구분한다.
- 모바일에서는 같은 세 탭을 화면 하단 고정 내비게이션으로 옮긴다.
- 모든 탭은 실제 `button`으로 만들고 현재 상태를 `aria-current`로 알린다.

### 홈

- 히어로를 진한 남색·파랑 그라데이션의 2열 카드로 구성한다.
- 문구는 `서울의 오늘을 더 가깝게 만나보세요.`를 유지한다.
- `축제 둘러보기`는 지역 탐색 화면으로 이동한다.
- `챗봇에게 물어보기`는 기존 챗봇 열기 동작과 연결할 수 있는 시각적 진입점으로 표시하되, 위젯 공개 인터페이스 변경이 필요하면 이번 단계에서는 안내 문구만 유지한다.
- 히어로 아래에는 지역정보 6,518건, 축제 201건, 데이터 유형 7개, 서울 자치구 25개를 통계 카드로 표시한다.
- 기능 바로가기는 `서울 축제 지도`, `익명 커뮤니티`, `Local AI` 세 카드로 구성한다.

### 지역 탐색

기존 `FestivalMapSection`의 필터와 지도 동작은 변경하지 않는다.

- 데스크톱: 왼쪽 필터 패널, 오른쪽 지도
- 태블릿·모바일: 필터가 지도 위에 표시되는 1열 구조
- 섹션 제목, 결과 건수, 초기화 버튼의 위계를 명확히 한다.
- 필터 패널은 흰색 카드, 지도는 독립된 큰 카드로 표시한다.
- 지도 팝업은 기존 정보와 동작을 유지하면서 색상과 여백만 통일한다.
- 오류와 결과 없음 문구는 기존 조건을 유지하고 파란색 디자인 시스템에 맞춘다.

### 커뮤니티

현재 작성 폼, 목록, 인라인 수정·삭제 흐름을 그대로 유지한다.

- 데스크톱: 왼쪽 작성 카드, 오른쪽 최근 글 목록
- 모바일: 작성 카드 다음에 목록이 나오는 1열 구조
- 작성 카드는 연한 파랑 배경 또는 파란 상단 강조선을 사용한다.
- 게시글은 제목, 날짜, 본문, 행동 버튼이 명확히 구분되도록 여백을 조정한다.
- 수정·삭제 패널은 같은 카드 내부의 별도 연한 배경 영역으로 구분한다.
- 기본 버튼은 파랑, 보조 버튼은 연한 회색, 삭제 확인 버튼은 오류색을 사용한다.
- 기존 성공·오류 메시지는 아이콘 없이 색상과 텍스트로 명확히 구분한다.

### 챗봇

기존 `FestivalChatWidget`의 동작은 변경하지 않고 CSS만 조정한다.

- 런처와 헤더는 기본 파랑을 유지한다.
- 대화창 배경, 말풍선, 근거 카드의 반경과 그림자를 공통 토큰에 맞춘다.
- 사용자 말풍선은 파랑, 챗봇 말풍선은 흰색을 유지한다.
- 모바일 전체 화면, 안전 영역, z-index 동작을 보존한다.
- 기존 로딩, 빠른 질문, 오류, 근거 카드 표시 조건을 변경하지 않는다.

## 상태와 데이터 흐름

새로운 도메인 데이터 흐름은 추가하지 않는다.

```text
내비게이션 클릭
-> App activeView 변경
-> location.hash 갱신
-> HomeView / ExploreView / CommunityView 중 하나 표시
-> 지도·커뮤니티 내부 상태와 챗봇 상태는 기존 로직 사용
```

화면 전환 외에는 API 요청, localStorage 저장, 지도 필터, 챗봇 요청 형식을 변경하지 않는다.

## 접근성과 반응형

- 키보드 포커스는 `:focus-visible`에서 3px 파란 외곽선으로 표시한다.
- 탭과 버튼의 터치 영역은 최소 44px로 유지한다.
- 선택 탭은 색상과 `aria-current`를 함께 사용한다.
- 모바일 하단 내비게이션 때문에 본문과 챗봇 런처가 가려지지 않도록 safe-area 여백을 확보한다.
- 900px 이하에서 지도와 커뮤니티를 1열로 변경한다.
- 640px 이하에서 기존 챗봇 전체 화면 동작을 유지한다.
- `prefers-reduced-motion: reduce`에서 스크롤·hover 이동 효과를 제거한다.

- 기존 커뮤니티 localStorage 테스트 통과
- 기존 축제 검색·챗봇 Function 테스트 통과
- TypeScript 검사 통과
- Vite 프로덕션 빌드 통과
- Netlify Functions 번들 통과

### 시각 검증

- 데스크톱 `1440×900`: 홈·탐색·커뮤니티·챗봇
- 태블릿 `768×1024`: 지도와 커뮤니티 1열 전환
- 모바일 `390×844`: 하단 탭, 챗봇 전체 화면, 입력 폼
- 긴 게시글 제목·본문, 빈 목록, 오류 메시지, 지도 결과 없음 상태
- 키보드 탭 순서와 focus-visible
- reduced-motion 상태

## 구현 순서

1. 현재 미커밋 기능 변경과 테스트를 보존하고 기준 상태를 기록한다.
2. 공통 디자인 토큰과 전역 포커스·배경을 적용한다.
3. SiteHeader와 세 View로 App 레이아웃을 분리한다.
4. 홈 화면을 현재 파란색 디자인으로 구성한다.
5. 지도 관련 컴포넌트의 레이아웃과 스타일만 개선한다.
6. 커뮤니티의 기존 템플릿 구조와 로직을 유지하며 스타일만 개선한다.
7. 챗봇 동작을 유지하며 스타일만 통일한다.
8. 자동 검증과 세 가지 뷰포트 시각 검증을 수행한다.
