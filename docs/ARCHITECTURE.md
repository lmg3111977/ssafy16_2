# 챗봇 아키텍처

## 요청 흐름

```text
사용자
  -> Vue 3 FestivalChatWidget
     - 최근 메시지 4개
     - 직전 출처 ID 5개
     - 공개 커뮤니티 스냅샷 최대 20개
  -> POST /api/chat
  -> Netlify Function
     - 입력과 공개 필드 재검증
     - 질문 라우팅
       - out-of-scope: 즉시 안내
       - project-chat: 검색 생략
       - project-data: 관련 제공자만 검색
     - 제공자별 최대 3건, 전체 최대 5건
     - OpenAI Responses API
  -> 답변과 공통 근거 카드
```

## 검색 공급자

`localhub-data.ts`가 제공 JSON을 공통 인덱스로 정규화하고 warm invocation 동안 캐시합니다. `localhub-search.ts`는 라우터가 고른 종류만 검색합니다.

| 유형 | 건수 |
|---|---:|
| 관광지 | 783 |
| 문화시설 | 566 |
| 레포츠 | 126 |
| 숙박 | 423 |
| 쇼핑 | 4,368 |
| 여행 코스 | 51 |
| 축제·공연·행사 | 201 |
| 합계 | 6,518 |

쇼핑처럼 큰 공급자가 결과를 독점하지 않도록 공급자당 3건을 먼저 자른 뒤 교차 병합합니다. 축제 검색은 `YYYY년 M월`, `M월에`, 오늘, 내일, 이번 주, 주말, 이번 달, 다음 달, 자치구 조사, 무료 조건을 별도로 해석합니다.

## 속도 결정

- JSON 전체를 LLM에 보내지 않음
- 범위 밖 질문은 LLM 호출 안 함
- 프로젝트 일상대화는 검색 안 함
- 모델 `gpt-5.6-luna`, `reasoning.effort: none`
- 출력 800토큰
- OpenAI 12초·재시도 1회, 브라우저 15초
- SSE 스트리밍은 현재 제외

## 보안 경계

브라우저에는 질문, 답변, 공개 근거, `/api/chat` 주소만 노출됩니다. `OPENAI_API_KEY`, 시스템 지침, OpenAI 호출은 Function에만 있습니다. 커뮤니티 문맥은 새 객체로 구성해 `id`, `title`, `content`, `createdAt`만 허용하며 비밀번호와 해시를 전달하지 않습니다.

## 커뮤니티 연동 제한

현재 게시판 Function은 번들 내부 `community.json`을 수정합니다. 이 파일 쓰기는 Netlify 배포 환경의 영구 저장 방식이 아니고, 별도 Function 번들과도 공유되지 않습니다. 그래서 챗봇은 실시간 게시판 전체를 서버에서 직접 읽지 않습니다. 운영 연동 전에는 영구 저장소를 도입하고 공개 조회 API를 공통 데이터 원천으로 사용해야 합니다.

## 장애 복구

API 키가 없거나 OpenAI가 시간초과·오류를 반환하면 검색 근거만으로 대체 답변을 만듭니다. 브라우저 네트워크 오류와 15초 시간초과에는 같은 질문을 다시 보낼 수 있는 버튼을 표시합니다.
